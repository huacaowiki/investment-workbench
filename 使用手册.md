# 投资研究工作台 · 使用手册

> 版本：2.0（2026-07-07）｜ 适用：Windows + Claude Code 桌面端 + OneDrive 双机同步 + GitHub 自动备份
> 你不需要任何编程背景。日常只用到「对 Claude 说一句话」或「复制一条命令回车」。

---

## 一、这套系统是什么

一个装在你 OneDrive 里、同时用 GitHub 做版本备份的**个人投资研究流水线**，
把《花茶个人投资体系 v4.1》变成了可执行的程序：

```
你的投资规则（config/ 四份YAML，铁则层，程序只读）
        ↓ 约束
数据抓取（akshare 免费公开接口：东财/新浪/巨潮/中证/同花顺，自动缓存、自动换备源）
        ↓ 喂给
分析引擎（一票否决 → 选股门槛 → 估值分级 → 风控参数，v4.2.0起全自动判定，逐条对照规则）
        ↓ 产出
标准化报告（output/ 自动归档、自动更新索引）
        ↓ 积累
体系迭代（月度/季度统计复盘 → 只出修订草案，改不改由你）
        ↓ 同步
每条命令跑完自动 git 提交并推送到 GitHub，两台电脑无缝同步（本手册第四节详解）
```

**四条底线**（写死在代码和项目指令里）：
1. 程序和 Claude 都**不能改** `config/` 里的规则——改规则永远是你手动操作
2. 判定全自动（v4.2.0）：无直接数据源的项按裁决的**替代量化口径**执行（如审计意见=
   近1年公告标题扫描无异常关键词→推定通过），推定口径在报告中逐条标注，绝不编造
3. 所有报告只做研究分析，**不构成买卖建议**
4. 每次运行命令后自动同步到 GitHub，**你不需要手动 commit/push**，也不需要每次告知 Claude 去推送

## 二、目录一图流

| 目录 | 放什么 | 你会不会动它 |
|---|---|---|
| `config/` | 你的投资规则（铁则） | 只在"确认修订草案"时手动改 |
| `templates/` | 报告模板 | 想换报告样式时改 |
| `knowledge/reference/` | 第三方研报资料 | 随时丢新资料进来 |
| `knowledge/drafts/` | 草稿、旧对话 | 随时丢，仅迭代时被读取 |
| `prompts/` | 一键工作流指令 | 不用动 |
| `src/`、`tests/` | 程序代码 | 不用动 |
| `output/` | 所有生成的报告 | 只读，别手改（meta供迭代统计用） |
| `data/` | 数据缓存与状态（市场状态/备选池/成交额历史） | 不用动；**不进Git**，靠OneDrive同步 |
| `archive/` | 原始素材备份 | 不用动 |

## 三、日常使用（一键工作流）

打开 Claude Code（项目目录 `investment-workbench`），直接说：

| 你说 | 发生什么 | 耗时 |
|---|---|---|
| **执行市场日报** | 生成当日市场日报到 `output/daily_market/` | 1-3分钟 |
| **分析 600519** | 生成该股全规则核对报告到 `output/stock_reports/` | 1-3分钟 |
| **执行月度迭代** | 统计当期报告，产出迭代报告+修订草案 | 1-2分钟 |
| **执行季度迭代** | 季度深度复盘，多三个专项分析 | 1-2分钟 |

不想开 Claude 时，也可以在项目目录打开终端直接敲命令（效果完全一致）：

```bash
python run.py daily              # 市场日报（当日）
python run.py daily 20260401     # 市场日报（指定历史日期）
python run.py stock 600519       # 个股分析
python run.py monthly            # 月度迭代（上个自然月）
python run.py monthly 2026-06    # 月度迭代（指定月份）
python run.py quarterly          # 季度迭代（上个自然季度）
```

> 每条命令跑完，终端会打印一行 `🔄 Git同步：已提交并推送到 GitHub`——这就是自动同步生效的标志，
> 你不需要额外做任何事。详见第四节。

### 报告格式（v4.3.0：默认HTML，Claude桌面端风格）

日报与个股报告默认生成 **HTML**（浏览器双击打开，样式内嵌离线可用）+ MD（机器可读源文件，
永远落盘）。需要其他格式时加 `--format`：

```bash
python run.py stock 600519                  # 默认：HTML + MD
python run.py stock 600519 --format md      # 仅 MD
python run.py stock 600519 --format all     # MD + HTML + PDF（Edge无头打印，与HTML视觉一致）
python run.py daily --format pdf            # MD + HTML + PDF
```

三种格式内容完全一致；PDF自动分页（表格不跨页断裂）、每页页脚含报告名与生成日期。
本机没有Edge时PDF自动跳过并提示（HTML/MD不受影响）。

### 多锚估值怎么读（v4.3.0 个股报告第五节）

每只股票按行业自动匹配 3 种估值方法（核心锚/辅助锚/验证锚，如金融=PB分位+股息锚+同业PE，
成长=PE分位+PEG+PB分位），逐一给出测算过程与区间：

- **方法间偏差≤30%** → 输出加权的「综合合理区间 / 安全边际价 / 高估阈值」及现价位置
- **偏差>30%** → 不强行加权，分别列出各方法区间并解释分歧来源（这本身就是重要信号）
- 某方法输入缺失（如亏损股PE法、不分红标的股息锚）→ 标注"不可用+原因"，不编造

注意：多锚区间**只是研究参考**，不改变体系的买卖门槛判定；你研究笔记里定义的估值锚
永远优先（报告结果可用来交叉验证你的锚）。

### 市场状态判定（v4.2.0 已自动化）

程序按 §1.3 的量化条件自动初判 A/B/C 区，生成日报时自动生效，无需你每周手动判断：

```bash
python run.py judge-state        # 手动触发程序初判并落盘（随时可跑，日报也会顺带触发）
python run.py set-state B_high   # 人工判定：7日内优先于程序初判（周日想自己复核时用）
```

自动初判口径：A/C 区各按可判定量化条件满足**≥3条**进入；否则按股债利差
（1/中证全指PE − 10年国债收益率）落 B 区细分带。定性条件（政策表述、新股民开户数、
偏股基金日光基、全市场破净率）无公开数据源，不参与自动判定，仅在日报第五节列示供你参考。

### 择时备选池（watchlist）

§2.3"理解度"门槛已自动化为备选池成员资格——你把研究过的标的加入池子，
程序即视为完成研究义务（入池时会提醒研究笔记要求）：

```bash
python run.py watchlist list          # 查看备选池（目标30-50只，覆盖≥5行业）
python run.py watchlist add 002594    # 入池
python run.py watchlist remove 002594 # 出池（季度审核不达标时）
```

不在池内的标的，择时组合"标的必须在备选池内"门槛直接判 FAIL。

## 四、双电脑无缝同步（Git 自动推送 + OneDrive 实时同步）

系统用**两条互补的同步链路**，你不需要在两台电脑之间手动搬运任何文件：

| 同步链路 | 负责什么 | 触发方式 | 延迟 |
|---|---|---|---|
| **Git（GitHub）** | 代码、config规则、报告、模板、文档等**版本化**内容 | 每条 `run.py` 命令跑完自动触发；Claude 编辑代码后也会自动提交推送 | 秒级（需联网） |
| **OneDrive** | `data/` 下的缓存与状态文件（市场状态、备选池、成交额历史等，体积小、变化频繁、无需版本历史） | 系统后台实时同步 | 通常秒级 |

### 4.1 两台电脑都要做的一次性准备

1. 两台电脑都登录同一 OneDrive，确认 `investment-workbench` 文件夹完全同步
2. 两台电脑都要能推送到 GitHub 私有仓库 `huacaowiki/investment-workbench`：
   - 装 [GitHub Desktop](https://desktop.github.com/) 并登录同一账号，或
   - 命令行装 [GitHub CLI](https://cli.github.com/)，执行 `gh auth login` 完成一次授权
   - 确认能推送：在项目目录执行 `git remote -v`，应看到 `origin` 指向该仓库
3. 装 Python（≥3.10，勾选 Add to PATH）→ 项目目录终端执行 `pip install -r requirements.txt`

做完这两步，之后**两台电脑上的每一次操作都会自动同步**，无需再手动 `git pull` / `git push`
（但如果你间隔很久没用某台电脑，重新打开时程序会在下次运行命令时自动 `pull --rebase` 拉齐最新版本）。

### 4.2 自动同步实际发生了什么

每次 `python run.py <命令>` 跑完，程序自动执行：

```
git add -A          # 把这次产生的变更（报告、config历史备份等）全部纳入
git commit -m "..."  # 自动生成提交信息
git pull --rebase --autostash   # 先拉取另一台电脑可能推送过的更新，避免分叉
git push             # 推送到 GitHub
```

Claude 在这个项目里直接帮你改代码、改文档时，也会遵循同样的流程主动提交推送——
**你不需要说"帮我推送一下"**，这已经是标准操作，写在了项目指令（`CLAUDE.md`）里。

如果你想临时跳过自动同步（比如完全离线环境），在命令后加 `--no-sync`：

```bash
python run.py daily --no-sync
```

### 4.3 同步状态怎么看

命令执行完，观察最后一行：

- `🔄 Git同步：已提交并推送到 GitHub` → 一切正常
- `⚠️ Git同步：已提交但推送失败（稍后可手动 git push）` → 本地已保存，只是暂时没联网/远程有冲突，
  换个网络环境后手动执行 `git push` 补上即可
- 没有任何 Git同步提示 → 说明这次命令没有产生任何文件变更（比如 `watchlist list` 纯查看），
  属于正常情况，无需担心

### 4.4 出现冲突怎么办

正常使用下几乎不会遇到（两台电脑很少同时改同一个文件），万一出现 `pull --rebase` 冲突提示：

1. 不要慌，本地的改动不会丢——冲突文件里会同时保留两边的版本标记
2. 打开 Claude Code，直接说「帮我看一下 git 冲突，解决一下」，Claude 会读取冲突文件、
   结合上下文帮你合并（config/ 冲突时会额外提醒你人工确认哪个版本的规则是对的）
3. 解决后正常运行任意 `run.py` 命令即可恢复自动同步

## 五、修改投资体系配置（唯一正确流程）

1. **先备份**：`python run.py backup-config 本次修改原因`
2. 用任意文本编辑器打开 `config/` 对应 YAML，修改并保存（注意保持缩进格式）
3. 验证没改坏：`python -m pytest tests/ -q`（全绿即OK；报错说明YAML格式坏了）
4. 在 `meta.changelog`/`confirmed_decisions` 里加一行版本记录
5. 改坏了想撤销：`python run.py rollback <版本号>`（版本号用 `config/history/manifest.json` 里查）
6. 保存后正常运行任意 `run.py` 命令，改动会自动同步到 GitHub 和另一台电脑

> 纪律提醒（§4.3）：参数调整需≥6个月数据回顾；同一参数一年最多改2次；重大迭代冷静一个月。

## 六、知识库资料更新

- 新研报/文章 → 按内容丢进 `knowledge/reference/` 的 `macro_policy`（宏观）/
  `industry_research`（行业与个股）/ `ai_industry`（AI产业）
- 自己的碎片想法、复盘笔记 → `knowledge/drafts/personal_notes/`
- 旧 Claude 项目导出 → `knowledge/drafts/old_projects/`
- 不用改名、不用排版；月度迭代会自动列出当期新增清单
- 这些文件同样会被 Git 自动同步跟踪（下次跑任意 `run.py` 命令时一并提交推送）

## 七、体系迭代操作流程

1. 每月最后一个交易日后：对 Claude 说「执行月度迭代」
2. 打开 `output/iteration/` 的迭代报告，逐条看「配置修订草案」
3. 对每条草案做决定：**采纳 / 拒绝 / 继续观察**
4. 采纳的：按第五节流程改 config（先 `backup-config`！）
5. 季度末用「执行季度迭代」，多出三个深度专项；收益匹配度需要你把券商账户的
   季度收益率/最大回撤填进报告对应位置（系统不接入实盘账户）

## 八、常见问题排查

| 现象 | 原因 | 解决 |
|---|---|---|
| 报告里很多"数据缺失" | 数据源接口临时抽风 | 程序已自动重试+换备源；过几分钟重跑，或接受缺失（不影响其他判定） |
| `ModuleNotFoundError` | 没装依赖 | 项目目录执行 `pip install -r requirements.txt` |
| 命令没反应/极慢 | 首次拉数无缓存，或网络慢 | 等待2-3分钟；当日第二次跑会命中缓存秒出 |
| 中文乱码 | Windows 终端 GBK 编码 | 报告文件本身是好的（UTF-8）；只影响终端显示 |
| YAML 改完程序报错 | 缩进/冒号格式坏了 | `python run.py rollback <版本号>` 回滚，重新改 |
| 想换报告样式 | — | 直接改 `templates/` 下的模板（保留 `{占位符}` 即可） |
| 报告说"市场状态未判定" | 初判数据缺失 | `python run.py judge-state` 重跑；仍缺失则 `set-state` 人工判定 |
| 择时门槛"备选池"判FAIL | 标的未入池 | `python run.py watchlist add <代码>`（先完成研究笔记） |
| 成交额连续性条件样本不足 | 序列随每日运行累积 | 坚持每天跑一次日报，10个交易日后自动收敛 |
| 提示"Git同步：推送失败" | 网络问题或远程有新提交未拉取 | 检查网络后手动 `git pull --rebase` + `git push`，或让 Claude 处理 |
| 两台电脑内容不一致 | 某台电脑离线太久 / 用了 `--no-sync` | 联网后随便跑一条命令即可自动拉齐；也可手动 `git pull` |
| 想彻底手动接管 Git | — | 直接用你熟悉的 `git`/GitHub Desktop 操作即可，自动同步不会与手动操作冲突 |
