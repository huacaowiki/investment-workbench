# 主控指令：执行个股分析

## 使用示例（直接说其中一句即可）
- 「分析 600519」/「分析比亚迪 002594」→ 生成11模块深度报告（默认HTML看板+MD）
- 「分析 600519，出PDF」→ 对应 `python run.py stock 600519 --format all`
- 报告含：综合评级裁决区（评分/评级/仓位自动联动config）、选股校验、技术面、
  基本面、多锚估值、多空交锋、分级风险、实操方案、验证节点

> 触发方式：用户在对话中说「分析 <股票代码>」（如「分析 600519」「分析比亚迪 002594」）

## 执行链路（严格按序）

1. **确认代码**：从用户输入提取6位股票代码（支持 sh/sz 前缀与 .SH/.SZ 后缀自动规整）
2. **运行分析**：`python run.py stock <代码>`
   - 该命令自动完成：拉取全量个股数据（基本面/行情/估值分位/财务/分红/质押/公告/同业）
     → 一票否决检查 → 股息组合门槛（通用+分类）→ 择时组合门槛与评分
     → 估值分级 → 风控参数测算 → 渲染归档到 output/stock_reports/
3. **输出摘要**：向用户汇报——
   - 报告路径与综合结论（核心池/观察池/排除）
   - 关键判定：哪些门槛 PASS / FAIL / 数据缺失（v4.2.0 全自动判定，无人工项）
   - 推定类判定的口径提示（如审计意见=公告扫描推定），指向报告第八节
   - 若标的不在备选池且用户有意跟踪：提示 `python run.py watchlist add <代码>`
   - 提醒：若考虑买入，必须填写决策卡片（§2.6），择时标的另有24小时冷静期

## 输出要求

- 打分与估值结果 100% 以程序输出为准，禁止心算修改
- 若用户追问"能不能买"：重申本系统只做研究分析与规则核对，不提供买卖建议；
  指出体系内的下一步动作（入池→决策卡片→冷静期）
- 若标的行业初判为能力圈外（excluded）：明确提示§2.1禁止建仓红线，且归类需人工最终确认

## 注意事项

- **禁止**修改 config/；**禁止**读取 knowledge/drafts/
- knowledge/reference/ 中如有该标的研报（如BYD、腾讯），可引用作背景论据，
  但必须标注来源，且不得覆盖程序判定结论（参考级 < 铁则层）
- 港股/美股代码：当前数据层仅支持A股，如实告知并建议人工分析流程
