# 主控指令：执行市场日报

> 触发方式：用户在对话中说「执行市场日报」（或指定日期，如「执行市场日报 20260401」）

## 执行链路（严格按序）

1. **拉取数据**：运行 `python run.py daily`（指定日期则 `python run.py daily YYYYMMDD`）
   - 该命令自动完成：拉取当日大盘/板块/涨跌停/沪深港通/龙虎榜/波动率/国债收益率
     → 市场分析（对照 config 铁则）→ 渲染模板 → 归档到 output/daily_market/
2. **核对输出**：确认报告文件已生成、INDEX.md 已更新、meta JSON 已写入
3. **输出摘要**：向用户汇报——
   - 报告路径
   - 大盘指数涨跌与两市成交额
   - 情绪观察分（说明这是工程化观察指标，非铁则）
   - 市场状态（v4.2.0：程序自动初判生效；人工 set-state 7日内优先。汇报生效状态与来源）
   - P0/P1 级风险提示（如有）

## 输出要求

- 摘要中的一切结论必须与报告一致，不额外发挥
- 涉及【待确认】口径（如波动率近似算法）时保留口径标注
- 数据缺失项如实告知，不掩饰

## 注意事项

- **禁止**修改 config/ 下任何文件
- **禁止**读取 knowledge/drafts/（本工作流非迭代场景）
- knowledge/reference/ 资料可用于补充背景，但不得改变判定结论
- 命令失败时：查看报错 → 修复环境问题（如依赖缺失 `pip install -r requirements.txt`）→ 重试；
  数据源全挂时报告仍会生成（缺失项标注），如实汇报即可
- 报告免责声明不得删除
