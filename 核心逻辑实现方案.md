## 一、选股评级联动（任务书模块1/3 的规则底座）

### 1.1 综合评分体系（新增 config/stock_selection.yaml `composite_scoring` 段，v4.4.0）

体系原文是"门槛制+计数制"，无百分制评分。按任务书授权新增**展示层评分映射**（不改变门槛判定本身，
门槛FAIL仍直接排除）；全部权重写入config，代码读配执行，每项得分标注计算依据：

| 维度 | 分值 | 计分项（来源=既有自动判定结果） |
|---|---|---|
| 财务质量 | 40 | ROE门槛(12)、净利润结构(10)、资产负债率(6)、收现比(6)、净现比(6)；金融类重分配：ROE(18)+净利结构(12)+分红趋势(10) |
| 分红与股东回报 | 20 | 连续分红年数(8)、派现比例(6)、股息率门槛(6) |
| 流动性与规模 | 10 | 市值门槛(5)、成交额门槛(5) |
| 择时信号 | 15 | 4项自动评分（技术底部/右侧/资金/机构预期）×3.75 |
| 估值位置 | 15 | PE或PB历史分位：<30%=15；30-50%=10；50-70%=5；≥70%=0；样本不足=0并标注 |

计分规则：PASS=满分、FAIL=0、MISSING=0（缺失不给分，偏保守），每项在报告中列示"得分/满分+依据"。

### 1.2 评级与仓位推导（新增 `rating_mapping` 段）

按任务书映射 + 体系硬约束叠加（**任何评级不得越过铁则**）：

```
前置门槛（任一命中→评级=排除，不进入评分映射）：
  一票否决FAIL / 能力圈excluded / ST·退市 / 上市不满1年（新股，任务书授权新增否决）
评级映射：
  得分≥80 且 估值定性=低估          → 买入
  得分60-79 且 估值定性∈{低估,合理} → 逢低建仓
  得分<60 或 估值定性=高估          → 观望
体系约束叠加（优先级高于映射）：
  市场状态=C区 → 评级封顶"观望"（§1.4只卖不买）
  市场状态=B偏高 → 买入降级为逢低建仓（暂停买入/不开新仓口径）
仓位：买入=单票上限×能力圈系数；逢低建仓=上限×系数×50%；观望/排除=0
  单票上限恒取 risk_control.yaml position_caps（不得突破，self_check双重校验）
```

估值定性=多锚综合区间位置（低于安全边际价=低估/区间内=合理/高于高估阈=高估）；多锚不可用时
回退历史分位口径（<30%低估 / 30-70%合理 / ≥70%高估）并标注回退。
盈亏比=(目标价-现价)/(现价-止损价)，目标=综合区间上沿，止损=现价×0.85（体系-15%硬止损）。
信心水平=数据完备度工程口径（可用估值方法数、MISSING项数、分位窗口是否降级 → 高/中/低，标注非主观判断）。

## 二、技术面分析引擎（新增 src/analyzer/tech_analysis.py）

- 数据：日线收/高/低/量序列（数据层快照新增近130日K线序列字段，缓存复用）
- 六维度：均线（MA5/20/60多空排列）、MACD（DIF/DEA/柱方向）、KDJ(9,3,3)、BOLL(20,2)位置、
  量能（5日均量/20日均量比）、趋势综合（六信号多空计数→偏多/偏空/纠缠）
- 关键价位表：支撑=MA20/MA60/BOLL下轨/60日低/52周低按距离分档；阻力=BOLL上轨/60日高/52周高，
  每档附来源逻辑；全部程序计算
- 免责口径：模块头标注"技术指标为量化描述，不构成走势预测"

## 三、日报核心修复与新增

1. **板块榜单修复**：涨幅榜=涨跌幅>0的板块降序取Top10，不足10个如实列示+标注"今日上涨板块不足10个"；
   跌幅榜=涨跌幅<0升序（跌最深在前）Top10，同样不凑数
2. **情景预判**：σ=上证近20日日收益率标准差；乐观=[+0.5σ,+2σ]、中性=[-σ,+σ]、悲观=[-2σ,-0.5σ]
   折算点位区间；概率固定框架25/50/25，全部标注"波动率推导的观察框架，非预测"
3. **核心矛盾**：多头证据（上涨板块数/两融变化/利差位置/涨停数）vs 空头证据（下跌结构/跌停/波动率/逼近状态切换线），逐条带数据
4. **极端行情标注**：涨停+跌停>200 或 单边>100 → 头部黄条"今日行情极端，数据参考性下降"
5. **非交易日兜底**：指数快照为空或当日无成交数据 → 输出"今日非交易日/数据未开盘"提示并退出，不生成空报告
