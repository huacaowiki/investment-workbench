## 一、目录结构说明

```
investment-workbench/
├── config/                     # 铁则层：唯一执行标准（禁止程序修改）
│   ├── investment_system.yaml  #   投资理念总纲、两组合框架、能力圈、市场状态判定
│   ├── stock_selection.yaml    #   一票否决 + 股息组合门槛(A/B/C类) + 择时门槛与评分 + ETF规则
│   ├── valuation_model.yaml    #   估值买入/卖出规则、价格区间分级、估值锚口径
│   ├── risk_control.yaml       #   仓位映射、建仓批次、止盈止损、硬性约束、迭代约束
│   └── history/                #   配置历史版本备份（version_manager 管理）
├── templates/                  # 报告模板（阶段三生成标准模板，可替换为自有模板）
├── knowledge/
│   ├── reference/              # 参考级：第三方资料（只能作论据，不得改变规则）
│   │   ├── macro_policy/       #   市场日报截图、M1相关性、金价历史图
│   │   ├── industry_research/  #   BYD×2、腾讯×2 分析报告PDF
│   │   └── ai_industry/        #   （空，后续放AI产业报告）
│   └── drafts/                 # 禁用级：仅体系迭代场景可读
│       ├── personal_notes/     #   （空，放碎片思考、复盘笔记）
│       └── old_projects/       #   （空，放旧Claude项目对话导出）
├── src/                        # 代码核心层（阶段二、三、四开发）
├── prompts/                    # 一键主控指令（阶段五交付）
├── output/                     # 报告归档：daily_market / stock_reports / iteration
├── data/                       # 本地缓存（.gitignore 已忽略，样例数据除外）
├── tests/                      # 单元测试与集成测试
├── archive/原始素材备份/        # 原始 00/01/02 三个文件夹整体归档留存
├── CLAUDE.md                   # 项目自定义指令（已替你完成设置，Claude Code 自动加载）
└── .gitignore
```

**关于项目自定义指令**：Claude Code 桌面端会自动读取项目根目录的 `CLAUDE.md` 作为项目级指令，效果等同于设置面板中的「项目自定义指令」。你无需再手动进设置粘贴（如仍想在设置面板粘贴一份，内容以 CLAUDE.md 为准）。
